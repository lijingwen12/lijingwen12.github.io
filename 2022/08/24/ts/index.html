定义希望被编译文件所在的目录
默认值：[“**/*”]

定义需要排除在外的目录
默认值：[“node_modules”,”bower_components”,”jspm_packages”]
示例：

extends

定义被继承的配置文件
示例：

files

指定被编译的列表，只有需要编译的文件少时才会用到
示例：

compilerOptions

编译选项是配置文件中非常重要也比较复杂的配置选项
在 compilerOptions 中包含多个子选项，用来完成对编译的配置

target

设置 ts 代码编译的目标版本
可选值：ES3（默认）、ES5、ES6/ES2015、ES7/ES2016、ES2017、ES2018、ES2019、ES2020、ESNext

lib

指定代码运行时所包含的库（宿主环境）
可选值：ES5、ES6/ES2015、ES7/ES2016、ES2017、ES2018、ES2019、ES2020、ESNext、DOM、WebWorker、ScriptHost ……

module

设置编译后代码使用的模块化系统
可选值：CommonJS、UMD、AMD、System、ES2020、ESNext、None

outDir

编译后文件的所在目录
默认情况下，编译后的 js 文件会和 ts 文件位于相同的目录，设置 outDir 后可以改变编译后文件的位置

outFile

将所有的文件编译为一个 js 文件
默认会将所有的编写在全局作用域中的代码合并为一个 js 文件，如果 module 制定了 None、System 或 AMD 则会将模块一起合并到文件之中

rootDir

指定代码的根目录，默认情况下编译后文件的目录结构会以最长的公共目录为根目录，通过 rootDir 可以手动指定根目录

allowJs

是否对 js 文件编译

checkJs

是否对 js 文件进行检查

removeComments

是否删除注释
默认值：false

noEmit

不对代码进行编译
默认值：false

sourceMap

是否生成 sourceMap
默认值：false

严格检查

  • strict

    • 启用所有的严格检查，默认值为 true，设置后相当于开启了所有的严格检查

  • alwaysStrict

    • 总是以严格模式对代码进行编译

  • noImplicitAny

    • 禁止隐式的 any 类型

  • noImplicitThis

    • 禁止类型不明确的 this

  • strictBindCallApply

    • 严格检查 bind、call 和 apply 的参数列表

  • strictFunctionTypes

    • 严格检查函数的类型

  • strictNullChecks

    • 严格的空值检查

  • strictPropertyInitialization

    • 严格检查属性是否初始化

  • 额外检查

    • noFallthroughCasesInSwitch
      • 检查 switch 语句包含正确的 break
    • noImplicitReturns
      • 检查函数没有隐式的返回值
    • noUnusedLocals
      • 检查未使用的局部变量
    • noUnusedParameters
      • 检查未使用的参数

  • 高级

    • allowUnreachableCode
      • 检查不可达代码
      • 可选值：
        • true，忽略不可达代码
        • false，不可达代码将引起错误
    • noEmitOnError
      • 有错误的情况下不进行编译
      • 默认值：false

字面量 (一般不使用)

有时候可以这么使用
使用 | 来连接多个类型(联合类型)

any | unknown

any 表示的是任意类型
不建议使用,相当于关闭 TS 类型检测
同时，声明变量不指定类型,TS 解析器会自动判断它为 any

如果遇到不确定类型的变量的需求,可以使用 unknown

unknown 和 any 作用类似,都可以任意赋值
区别：any可以赋值给任意类型的变量,unknown则只能赋值给unknown类型的变量
此时如果使用any,赋值给其他类型的变量,会出现类型改变的效果,所以不建议使用

如果想把unknown赋值给指定类型的变量,有以下几种方式

  1. 判断类型
  1. 类型断言
    注：可以用来告诉解析器实际类型，前提是你确定它就是 string, 有两种写法

void | never

void用来表示空，一般用来表示函数没有返回值

never用来表示永远不会返回结果
比如抛错，程序抛错，不会进行下去，所以不会有返回结果

三、面向对象

面向对象：就是程序之中所有的操作都需要通过对象来完成。
举例来说：

  • 操作浏览器要使用 window 对象
  • 操作网页要使用 document 对象
  • 操作控制台要使用 console 对象
    一切操作都要通过对象，也就是所谓的面向对象。

类（class）

定义类：
使用class关键字

使用

如果属性/方法使用关键字static修饰,则是类属性/方法
可以直接使用类访问

关键字readonly表示只读属性

构造函数和 this

构造函数关键字constructor
可以理解为 new Person() 的操作就是在调用 constructor
constructor里面的this指向你创建的对象（如下的 obj）
可以通过this向新建的对象添加属性

继承

定义一个表示动物的类
再定义一个狗的类，然后可以使用extends关键字继承动物类，拥有动物类的属性和功能
如果子类创建了与父类相同的方法和属性，是重写属性和方法，不会影响父类
super关键字：可以使用它来调用父类的方法和属性
如果子类中写了构造函数，在子类构造函数中必须对父类的构造函数进行调用，不然会相当于重写父类的构造函数，使得父类在构造函数中从操作丢失，而且也会报错，

抽象类

以关键字abstract开头的类是抽象类
抽象类和其他的类区别不大，只是不能用来创建对象
抽象类就是专门用来被继承的类

抽象类可以添加抽象方法
也是使用关键字abstract定义，没有方法体
抽象方法只能定义在抽象类中，子类必须对抽象方法进行重写，如果没有进行重写，会报错！

接口

接口用来定义类结构
用来定义一个类中应该包含哪些属性和方法
同时接口也可以当成类型声明去使用。
使用关键字interface定义

区别：接口可以重复声明，type 不可以，重复声明的接口，会合并起来

接口可以在定义类的时候去限制类的结构
接口中所有的属性都不能又实际的值
接口中定义对象的结构，而不考虑实际值
在接口中所有的方法都是抽象方法

定义类的时候，可以使用类去实现一个接口
实现接口就是使类满足接口的要求
关键字implements

接口主要的作用是定义一个标准，让类去符合一个标准

属性

public： 修饰属性，公共属性，所有都能访问
private：私有属性，私有属性只能在类内部访问
protected：受包含的属性，只能在当前类和当前类的子类中访问

TS 中有一个属性设置和获取的方法
关键字get、set
